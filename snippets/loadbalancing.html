What is Load Balancing?

Load balancing is the process of distributing incoming network or application traffic across multiple servers to ensure that no single server becomes overwhelmed. It helps to optimize resource use, maximize throughput, minimize response time, and ensure system reliability.

Benefits of Load Balancing

  • Improved Availability and Reliability
    Load balancers ensure high availability by routing traffic to healthy servers. If a server goes down, the load balancer redirects traffic to the other servers, preventing downtime.
  • Increased Scalability
    By distributing traffic across multiple servers, load balancers allow you to scale horizontally. As demand grows, you can add more servers, and the load balancer will spread the load, ensuring smooth performance.
  • Optimized Resource Utilization
    Load balancing helps in optimizing the utilization of servers. It ensures that each server is used efficiently without being overloaded, which helps in reducing costs and improving system performance.
  • Reduced Latency
    Load balancers can distribute traffic based on server location, which means users get served by the nearest server, resulting in lower latency and faster response times.


Types of Load Balancers

Network Load Balancers (NLB)
  • Operate at the transport layer (Layer 4)
  • Best for handling large volumes of TCP/UDP traffic, with a focus on speed and simplicity
  • Use Case: Ideal for applications that require ultra-fast throughput without much need for advanced routing decisions.
Application Load Balancers (ALB)
  • Operate at the application layer (Layer 7)
  • Allow for intelligent routing based on HTTP headers, path, and more, providing granular control over traffic.
  • Use Case: Ideal for web applications, API gateways, and services that need advanced request processing capabilities.

Load Balancing Algorithms

Load balancers use different algorithms to determine how to distribute incoming traffic:
Round Robin
Traffic is distributed in a circular order across all available servers. Simple but effective when servers have similar capabilities.
Least Connections
Traffic is directed to the server with the fewest active connections, optimizing for current resource use.
Weighted Round Robin
Servers are assigned weights based on their processing capacity, and traffic is distributed accordingly.
 
IP Hash
The IP address of the client is hashed, and traffic is consistently routed to the same server, useful for session persistence.
